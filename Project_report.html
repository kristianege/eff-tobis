2.1 Otolith preperation, imaging and measurement documentation

The following files contains information on how otolith were worked up.

  • Protocol_otolith_preparation.doc, documentation on how otoliths were prepared
  • Protocol_otolith_image&measurement.doc, documentation on imaging and measurements
  • Naming of images.doc, how images were named

I’ll just briefely note here that for each larvae if possible both otolith were measured. But in some cases one was lost. In each otolith dayli increments were measured in two radii, on in direction of fastest growth and one some other direction, usually the best visible alternative direction, however not in any systematic way.

The following code chunk runs the script manipulation-1-raw-larvae-otolith-data.R

3.1.2 R code

source("manipulation-1-raw-larvae-otolith-data.R")

The script treats the raw larval otolith readings raw_larvae_otolith_measurements.csv and produces the following:

    1. Removes wrongly marked otolith rings, the marks were reviewed in pictures using a Matlab script
    1. Transforms all distances from pixels to microns
    1. Recalculates distance between the otolith dayli rings after removal of wrongly red rings
    1. Ring width are filtered so that they are between 0.5 and 9 microns 5. Ringwidths bigger than 1.8 the following ring are excluded, this in order to sort out areas with no readings.

The resulting output dataframe is called oto2 also the raw data is maintained in the global environment as oto_raw both data frames are used in the next script manipulation-2-bestage-selection-and-more.R:

source("manipulation-2-bestage-selection-and-more.R")

This script is a bit more comprehensive and draws on several other scipts

  1. Removes readings in the outer 10% of the otolith to avoid any edge effect. Resulting data frame oto_no_edge

  2. Removes otoliths in which the predicted linear relationship between distance to center and the ring width does not pass through an interval of max and min ringwidth at 20 microns. The resulting data frame oto_inc_fit

  3. A linear model is then run on all each individual otolith radial reading. The model information is added to the data frame so that the data frame oto_filtered_model contains all model parameterisations for each red otolith radius.

  4. Model information as residuals, cooks distance, fitted values for each ringwidth is found in the data frame oto_filtered_all

  5. The script then sources three other scripts that are quite complex to explain, but the output is a selection of best ages for each fish. Where best age is selected on basis of highest number of daily increments in estimate, and secondary higher age this is done for both measured radii.

  6. Metadata as position etc. is added

  7. The two data frames oto_filtered_all and oto_filtered_model are then synced to include only the otoliths with “best ages”.

The output data frames are:
* oto4, containing all measurements after the above mentioned filtering
* oto5, containing the lm linear model parameters per filtered and selected larvae. These data frames are also available as csv’s otolith_filtered_all_marks and otolith_filtered in the project folder.

3.1.4 Resulting data frames

So after treatment of raw data and finding the linear relation between ring width and distance from center the following data frames are available:

  • oto_raw, all the larvae and raw data
  • oto2, treated measurements but before running linear model to find otolith growth, and before selecting according otoliths pr. fish according to “best ages”
  • oto4, all increment measurements after treatment and filtering, including model information. Only otoliths remaining after best ages selection .
  • oto5, containing linear model information from the relation between increment width and distance to center of otolith. Only otoliths remaining after best ages selection meta, with worked up metadata for further data exploration, same meta used for length measurements
  • d, see previous section for details
